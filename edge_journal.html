Google Drive setup — step by step
How it works
The app stores all your data in a single file called edge_journal_data.json saved directly in your Google Drive's private app folder. This folder is invisible to other apps and cannot be browsed in Drive — only this app can read it. Nothing is sent to any server except Google's own servers.
⏱ One-time setup, takes about 10 minutes. You will only do this once.
BEFORE YOU START
You need a Google account and the app must be opened from a URL (not a local file path like file://...). The easiest free option is to host the HTML file on GitHub Pages:
1. Create a free account at github.com
2. New repository → upload edge_journal.html → rename it to index.html
3. Settings → Pages → Source: main branch → Save
4. Your app URL will be https://yourusername.github.io/reponame
5. Open that URL in Chrome on your Android — bookmark it or Add to Home Screen
1
Create a Google Cloud project
Go to console.cloud.google.com
→ Click the project dropdown at the top (says "Select a project")
→ Click New Project
→ Name it anything, e.g. EdgeJournal
→ Click Create and wait a few seconds
→ Make sure the new project is selected in the dropdown
2
Enable the Google Drive API
In the left sidebar → APIs & Services → Library
→ Search for Google Drive API
→ Click the result → Click the blue Enable button
→ Wait for it to enable (page will reload)
3
Configure the OAuth consent screen
→ APIs & Services → OAuth consent screen
→ Select External → click Create
→ App name: Edge Journal
→ User support email: select your Gmail address
→ Developer contact email: your Gmail address
→ Click Save and Continue
→ On the Scopes page → click Save and Continue (no changes)
→ On Test users page → click + Add Users → add your own Gmail address → Save and Continue
→ Click Back to Dashboard
⚠ Keep the app in Testing mode — you do not need to publish it. Only the Gmail addresses you add as test users can sign in, which is exactly what you want for a personal app.
4
Create the OAuth Client ID
→ APIs & Services → Credentials
→ Click + Create Credentials → OAuth client ID
→ Application type: Web application
→ Name: anything, e.g. Edge Journal Web
→ Under Authorised JavaScript origins → click + Add URI
→ Paste your app's full URL, e.g. https://yourusername.github.io
  (no trailing slash, no path — just the origin)
→ Click Create
✓ A dialog will show your Client ID — it looks like
123456789-abcdefg.apps.googleusercontent.com
Copy it and paste it below.

TROUBLESHOOTING
"Error 403: access_denied"
→ You forgot to add your Gmail to Test Users in Step 3. Go back and add it.

"Error 400: redirect_uri_mismatch"
→ The URL in Authorised JavaScript origins doesn't exactly match the URL you're using. Check for trailing slashes or http vs https.

Popup is blocked
→ Chrome blocked the sign-in popup. Tap the address bar → site settings → allow popups for this site.

"This app isn't verified"
→ Click Advanced → Go to Edge Journal (unsafe). This warning appears because the app is in Testing mode — it's your own app, so it's safe.

Works on desktop but not Android
→ Make sure you added both the desktop and mobile URLs to Authorised JavaScript origins if they differ.
